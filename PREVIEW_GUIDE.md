# 미리보기 안내

이 문서는 초보자도 메리츠화재 김옥진 팀장 보험 상담 안내 랜딩페이지를 바로 확인할 수 있도록 만든 안내서입니다.

## 1. preview.html을 바로 여는 방법

`web` 폴더 안에 있는 `preview.html` 파일을 더블클릭하면 브라우저에서 바로 열립니다.

별도의 설치가 필요하지 않습니다.
`npm install`, `npm run dev`를 하지 않아도 화면을 확인할 수 있습니다.

## 2. React 개발 서버로 확인하는 방법

React + Vite + TypeScript 원본 프로젝트로 확인하려면 터미널에서 `web` 폴더로 이동해야 합니다.

```bash
cd meritz-kim-okjin-landing/web
```

그다음 패키지를 설치하고 개발 서버를 실행합니다.

```bash
npm install
npm run dev
```

터미널에 표시되는 주소를 브라우저에서 열면 됩니다.
보통 `http://localhost:5173/` 형태입니다.

## 3. npm이 없을 때 해결 방법

`npm` 명령을 찾을 수 없다는 메시지가 나오면 Node.js가 설치되어 있지 않거나 경로가 연결되지 않은 상태일 수 있습니다.

해결 방법:

1. Node.js LTS 버전을 설치합니다.
2. 설치 후 터미널을 완전히 닫고 다시 엽니다.
3. 아래 명령으로 설치 여부를 확인합니다.

```bash
node -v
npm -v
```

버전 숫자가 나오면 정상입니다.

## 4. npm run dev 실행 방법

개발 중 화면을 확인할 때 사용합니다.

```bash
cd meritz-kim-okjin-landing/web
npm install
npm run dev
```

개발 서버가 켜진 상태에서 코드를 수정하면 브라우저 화면이 자동으로 갱신됩니다.

## 5. npm run build 실행 방법

실제 배포용 파일을 만들 때 사용합니다.

```bash
cd meritz-kim-okjin-landing/web
npm run build
```

성공하면 `dist` 폴더가 생성됩니다.

## 6. 화면이 안 열릴 때 확인할 것

- `preview.html`을 여는 경우: 파일이 `web` 폴더 안에 있는지 확인합니다.
- React 개발 서버를 여는 경우: 터미널에 표시된 주소를 정확히 입력했는지 확인합니다.
- `npm run dev`가 실패하는 경우: `npm install`을 먼저 실행했는지 확인합니다.
- 브라우저 캐시 문제일 수 있으니 새로고침하거나 다른 브라우저로 열어 봅니다.
- 회사 보안 프로그램이나 방화벽이 로컬 개발 서버 접속을 막고 있지 않은지 확인합니다.

## 7. 실제 배포 전 수정해야 할 것

- `your-domain.com`: 실제 사용할 도메인으로 변경해야 합니다.
- `og-image.jpg`: 공유 이미지가 준비되면 실제 이미지 주소로 변경해야 합니다.
- 실제 문의 전송 기능: 현재 폼은 예시 메시지만 보여 주며 실제 전송되지 않습니다.
- 승인된 프로필 사진: 승인된 사진이 준비되면 플레이스홀더를 교체합니다.
- 승인된 로고 사용 여부: 공식 로고 사용 승인이 확인된 뒤에만 적용합니다.
- 준법 관련 고지 문구: 실제 배포 전 회사 내부 기준과 준법 검토에 맞게 확인합니다.
